Q1: Use Mathematical Induction to prove that : 1² + 2² + .......+ n² = n(n+1)(2n+1)/6

**Solution:** Step 1: Base Case (n = 1)

LHS = 1² = 1

RHS =

1(1 + 1)(2 × 1 + 1) / 6 = 1 × 2 × 3 / 6 = 6 / 6 = 1

Since LHS = RHS, the statement is true for **n = 1**.

Step 2: Induction Hypothesis

Assume the formula is true for n = k, i.e.,

1² + 2² + ... + k² = k(k + 1)(2k + 1) / 6

**Step 3: Inductive Step (For n = k + 1)**

We must prove:

$$1^2 + 2^2 + \cdots + k^2 + (k + 1)^2 = \frac{(k + 1)(k + 2)(2k + 3)}{6}$$

Using the induction hypothesis:

$$= \frac{k(k + 1)(2k + 1)}{6} + (k + 1)^2$$

Take (k + 1) common:

$$= (k + 1)\left[\frac{k(2k + 1)}{6} + (k + 1)\right]$$

Take LCM:

$$= (k + 1)\left[\frac{k(2k + 1) + 6(k + 1)}{6}\right]$$

$$= (k + 1)\left[\frac{2k^2 + k + 6k + 6}{6}\right]$$

$$= (k + 1)\left[\frac{2k^2 + 7k + 6}{6}\right]$$

Factorize:

$$2k^2 + 7k + 6 = (2k + 3)(k + 2)$$

So,

$$= \frac{(k + 1)(k + 2)(2k + 3)}{6}$$

This is the required RHS for $n = k + 1$.

(b) Getting at least a value 2

"At least 2" means ≥ 2

Favorable outcomes:
{2,3,4,5,6}

Number of favorable outcomes = 5

P(≥ 2) = 5/6

Q8: Describe the divide-and-conquer approach to solve recurrences ? Explain how this approach can be used to apply binary search in a sorted list.

**Solution:** 1. Divide-and-Conquer Approach

Divide-and-Conquer is an algorithm design technique that solves a problem by:

1. **Divide** - Break the problem into smaller sub-problems of the same type.
2. **Conquer** - Solve the sub-problems recursively.
3. **Combine** - Combine the solutions of sub-problems to get the final solution.

The running time of such algorithms is generally expressed using a **recurrence relation** of the form:

$$T(n) = aT\left(\frac{n}{b}\right) + f(n)$$

where:
* $a$ = number of sub-problems
* $n/b$ = size of each sub-problem
* $f(n)$ = time to divide and combine

2. Application to Binary Search

Binary Search works on a **sorted list** using divide-and-conquer.

Steps:
1. Compare the target element with the middle element.

3. If smaller → search in left half.
4. If larger → search in right half.

At each step, the problem size is reduced to n/2.

3. Recurrence for Binary Search

**Q17:** What is a bipartite graph ? Explain with the help of an example. Give one or two applications of bipartite graphs.

**Solution:** 1. Definition

A **bipartite graph** is a graph whose vertex set can be divided into two disjoint sets V₁ and V₂ such that:

*   Every edge connects a vertex from V₁ to a vertex in V₂.
*   No edge exists between vertices of the same set.

If every vertex of V₁ is connected to every vertex of V₂, it is called a **complete bipartite graph**, denoted by K<sub>m,n</sub>.

2. Example

Let:

V₁ = {u₁, u₂}, V₂ = {v₁, v₂, v₃}

Edges:

u₁ connected to v₁, v₂
u₂ connected to v₂, v₃

There are no edges within V₁ or within V₂.
Hence, the graph is bipartite.

A simple real-life example is:

*   Set V₁: Students
*   Set V₂: Projects
*   Edge: A student assigned to a project

*   Applications: scheduling problems, frequency assignment.
